# Release Notes — ASH v2.3.5

## Conformance Alignment

**First fully locked conformance release.** All 6 SDKs pass 134/134 vectors with byte-identical outputs generated from the Rust reference implementation.

| SDK | Vectors | Status |
|-----|---------|--------|
| Rust | 134/134 | Pass |
| Node.js | 134/134 | Pass |
| Python | 134/134 | Pass |
| Go | 134/134 | Pass |
| PHP | 134/134 | Pass |
| WASM | 134/134 | Pass |

Vectors are immutable for this `ash_version`. Governance rules: [`docs/conformance-governance.md`](../conformance-governance.md).

## Behavior Changes

These changes affect SDK users who depend on specific error codes or HTTP status values:

### Error Code / HTTP Status Alignment

All SDKs now return identical error codes and HTTP status values matching the Rust reference:

| Error | Code | HTTP Status |
|-------|------|-------------|
| Invalid JSON | `ASH_CANONICALIZATION_ERROR` | 484 (was 422 in Go/PHP) |
| Validation failure | `ASH_VALIDATION_ERROR` | 485 (was 400 in Go/PHP) |
| Mode violation | `ASH_MODE_VIOLATION` | 486 (was 400 in Go/PHP) |
| Missing scoped field | `ASH_SCOPED_FIELD_MISSING` | 475 (new) |

### Stricter Validation

- **Empty binding rejected.** `ashDeriveClientSecret` now returns `ASH_VALIDATION_ERROR` (485) for empty binding strings. Previously accepted silently in Go/PHP/WASM.
- **Timestamp format enforced.** No leading zeros (except `"0"`), digits only, seconds resolution.
- **Body hash format validated.** Must be exactly 64 lowercase hex characters.

### Canonicalization Fixes

- **Plus sign** (`+`) is literal in query strings, encoded as `%2B`. Previously inconsistent across SDKs.
- **Path normalization** decodes all percent-encoded characters (including `%2F` → `/`) then applies RFC 3986 dot-segment resolution.
- **Float formatting** aligned to ES6 `Number::toString` (RFC 8785 Section 7.2.2) across Go/PHP.

## Security Fixes

- **CRITICAL**: TOCTOU race condition in Redis stores (Python, PHP) — atomic Lua scripts
- **CRITICAL**: PHP proof verification bypass when nonce is null
- **HIGH**: Nonce generation enforced in all modes (Python, PHP)
- **HIGH**: Trusted proxy enforcement added to all SDKs
- **HIGH**: IP validation hardened (rejects 999.999.999.999)
- **HIGH**: CI/CD command injection via unsanitized workflow inputs

## New APIs

Go and PHP add strict/validated variants for conformance-aligned behavior:

- `AshNormalizeBindingStrict` — rejects invalid inputs (empty method, missing leading slash)
- `AshBuildProofHMACValidated` — validates all inputs before HMAC
- `AshHashProofValidated` — validates proof format
- `AshExtractScopedFieldsStrict` — fails on missing fields in strict mode
- `AshHashScope` — standalone scope hash computation

WASM adds new exports: `ashHashScope`, `ashExtractScopedFields`, `ashExtractScopedFieldsStrict`, with structured error objects `{ code, http_status, message }`.

## Schema Guard

- Conformance vectors locked to `ash_version: "2.3.5"` with `schema_version: 1`
- Vectors are immutable once published — any behavioral change requires version bump + regeneration
- Vector IDs are stable and never reused across versions
